# Firestore Database Design

Since Firestore is a NoSQL document database, the schema is designed around collections and sub-collections that optimize for the read patterns of the frontend application.

## 1. Collection: `users`
Stores user profiles and configuration settings.
- **Document ID**: `userId` (from Firebase Auth)
- **Fields**:
  - `name`: String
  - `email`: String
  - `dateOfBirth`: Timestamp
  - `baseHealthMetrics`: Object (e.g., weight, height, dietary restrictions)
  - `preferences`: Object
    - `exerciseIntervalHours`: Number (default: 8)
    - `fastingRoutine`: String (e.g., "16:8", "5:2")
  - `createdAt`: Timestamp
  - `updatedAt`: Timestamp

## 2. Collection: `events`
Stores user events that dictate contextual food suggestions.
- **Document ID**: Auto-generated
- **Fields**:
  - `userId`: Reference/String
  - `title`: String (e.g., "Marathon", "Wedding", "Yoga Day")
  - `date`: Timestamp
  - `eventType`: String (e.g., "endurance", "milestone", "wellness")
  - `description`: String

## 3. Collection: `daily_logs`
A dense log for each day per user containing their suggestions, tracking, and fasting progress.
- **Document ID**: `userId_YYYY_MM_DD`
- **Fields**:
  - `userId`: Reference/String
  - `date`: String ("YYYY-MM-DD")
  - `foodOfTheDay`: Object (Generated by Gemini)
    - `mealName`: String
    - `reasoning`: String (e.g., "Carb-loading for your upcoming marathon.")
    - `calories`: Number
    - `macronutrients`: Object
  - `fasting`: Object
    - `isActive`: Boolean
    - `durationLogged`: Number (hours)
  - `exercisesTriggered`: Array of Objects
    - `exerciseName`: String
    - `timeAssigned`: Timestamp
    - `completed`: Boolean
  - `healthCheckMetrics`: Object (daily logs of weight, water intake, etc.)

## 4. Collection: `health_analysis`
Stores periodic reports generated by Gemini summarizing the user's progress.
- **Document ID**: Auto-generated
- **Fields**:
  - `userId`: Reference/String
  - `weekStartDate`: Timestamp
  - `weekEndDate`: Timestamp
  - `aiSummary`: String
  - `trendScore`: Number (1-10)
  - `recommendationForNextWeek`: String
